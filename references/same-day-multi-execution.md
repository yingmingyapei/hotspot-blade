# 同一天多次热点刀锋执行：去重策略

## 问题场景

热点刀锋 cronjob 可能在同一天被触发多次：
- 定时任务自动执行（如 09:45）
- 手动 `hermes cronjob run` 触发补救执行
- Gateway 重启后补偿执行

当同一天已有执行记录时，第二轮必须选择**全新话题**，不能重复。

## 检测方法

```python
import json
from pathlib import Path

history_file = Path("/mnt/c/Users/yingm/wiki/sources/market-intelligence/daily/hotspot-blade-history.json")
with open(history_file) as f:
    hist = json.load(f)

today = "2026-05-30"
today_topics = []
for entry in hist.get("topics", []):
    if entry["date"] == today:
        today_topics = entry["topics"]
        break

if today_topics:
    print(f"⚠️ 今日已有 {len(today_topics)} 条执行记录: {today_topics}")
    print("→ 第二轮必须选择完全不同的 5 个话题")
else:
    print("✅ 今日首次执行")
```

## 去重规则

| 规则 | 说明 |
|------|------|
| **完全避开** | 今天已出现的话题绝对不可复用 |
| **子类轮换** | 如果首轮写了"国际"类 2 篇，第二轮优先选"社会/消费/科技" |
| **追加历史** | 第二轮的新话题追加到今天的 `topics` 和 `categories` 数组中 |
| **不覆盖** | 首轮的历史记录保留，仅追加 |

## Wiki 文件命名规则

同一日多次执行时，Wiki 文件使用轮次后缀区分，不覆盖首轮文件：

| 轮次 | 文件名 | 说明 |
|------|--------|------|
| 首轮 | `{date}-热点刀锋微头条-5篇.md` | 默认文件名 |
| 第二轮 | `{date}-热点刀锋微头条-R2.md` | R2 后缀 |
| 第N轮 | `{date}-热点刀锋微头条-R{N}.md` | 类推 |

**示例（2026-05-31）：**
- 首轮：`2026-05-31-热点刀锋微头条-5篇.md`
- 第二轮：`2026-05-31-热点刀锋微头条-R2.md`

历史 JSON 中的记录也追加到同一天条目，不创建新的 date 条目。

## 验证检查清单

```
□ 执行前读取历史文件，检查今天是否已有记录
□ 如果有记录，列出已用话题清单
□ 从热榜数据中筛选，排除所有已用话题（含近似话题）
□ 确认新选的 5 个话题与已用话题无重叠
□ 执行后将新话题追加到同日记录（不覆盖）
□ 在 Wiki 执行笔记中标注"今日第 N 轮执行"
```
