# 数据分析方法论（2026-06-04 实战总结）

## 核心原则

**数据分析不是公式打分，是用推理能力做判断。**

热榜数据就是一堆标题和热度值。没有经过分析的原始数据是噪音，不能直接用于选题。

## 分析五步法

### 第一步：通读所有原始数据
- 逐条阅读5个平台的全部热榜内容
- 不遗漏，不跳过，先看全貌
- 不限Token，充分阅读

### 第二步：识别社会矛盾和撕裂点
对每条热榜，问自己：
- 这个话题背后藏着什么社会矛盾？
- 哪些话题天然有两派对立？
- 哪些话题能嫁接到真实议题上？

### 第三步：交叉验证
- 同一个话题在多个平台同时热？→ 真正的热点，优先选
- 只在一个平台热？→ 可能是平台算法推的，谨慎
- 不同平台对同一话题的讨论角度有什么差异？

### 第四步：排除噪音
以下直接排除：
- 纯娱乐/饭圈/综艺/剧集
- 无争议的事实性新闻（地震/天气/赛事结果）
- 个人感悟/生活碎片
- 已过时效的话题（>3天）

### 第五步：从原始数据中发现爆点

**⚠️ 最关键的能力：不是照搬热榜标题，是从热榜数据里发现没人说过的角度。**

#### 案例：AI犯罪话题的发现过程

热榜原始数据：知乎 #19 "OpenAI遭起诉ChatGPT涉嫌帮策划校园枪击案"（标记：新）

表面分析：一个美国公司的法律新闻，跟中国读者关系不大
深度分析：
- 不是"OpenAI被起诉"，而是"AI已经能帮人策划犯罪了"
- 校园枪击案 + 汽车爆炸案 + 佛州起诉 → AI犯罪不是个案，是趋势
- 暗网上已有专门犯罪的AI模型 → 威胁已到家门口
- 中国孩子天天用的豆包、DeepSeek，安全吗？

**从"一个公司的法律新闻"提炼成"一个时代的安全问题"，这才是数据分析该做的事。**

#### 爆点发现清单

对每条热榜，问自己：
1. 这条新闻的表面是什么？（事实层）
2. 这条新闻背后藏着什么社会矛盾？（矛盾层）
3. 这个矛盾能嫁接到哪个一直存在的真实议题上？（议题层）
4. 如果我写这个角度，评论区会不会吵翻天？（验证层）

如果第4步答案是"会" → 选中
如果第4步答案是"不会" → 排除或换角度

逐条比对候选话题 vs 历史7天已写话题：
- 完全相同 → 排除
- 同一事件不同角度 → 降级，除非有重大新进展
- 同一社会议题但不同切入 → 可用，但需说明新角度
