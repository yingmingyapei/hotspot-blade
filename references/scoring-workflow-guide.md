# 选题评分实操指南（2026-06-15）

## 概述

选题评分是每日选题报告的核心步骤。从197条热榜数据中筛选5个候选话题，需要走完：全量评分→去重检查→交叉验证→手动调优 四个阶段。

## 阶段1：全量评分

所有候选话题按5维度打分，加权算总分。**必须用Python脚本批量评分**，不能手动逐条评判（数量太大）。

### 评分维度的客观判断标准

为了防止评分偏差，在每个维度上使用**关键词触发+调分**的结构化方法：

**钱包距离（1-10分，权重35%）**
- 9-10：匹配关键词 房价/房贷/工资/社保/养老金/物价/失业/裁员/涨薪/买房/租金
- 7-8：匹配关键词 黄金/金价/消费/手机/汽车/保险/医疗/教育/学费/补贴/降价/涨价/套餐
- 5-6：匹配关键词 经济/GDP/股市/关税/汇率/贸易/新能源/销量
- 3-4：匹配关键词 科技/AI/机器人/编程/数码
- 1-2：无以上关键词

**反驳成本（1-10分，权重25%，越高越好=反驳越容易）**
- 9：匹配 你/你的/我们/普通/老百姓/打工人/穷人/中产/年轻人（读者能直接代入自身经验反驳）
- 8：匹配 该不该/对不对/合理吗/公平/争议/如何看待（可表态）
- 7：匹配 价格/下跌/上涨/销量/暴跌/暴增（能用身边例子反驳）
- 5：匹配 国际/外交/政策/法规/规定（需要背景知识）
- 4：匹配 科技/AI/机器人/编程/技术/芯片（需要专业知识）

**物件锚点（1-10分，权重15%）**
- 9：匹配≥2个具体物件（手机/电脑/车/房子/黄金/工资条/外卖/股票/套餐等）
- 8：匹配1个具体物件
- 4-5：匹配 工程/项目/投资/产业/行业（抽象概念）
- 1-3：纯抽象，无物件

**头条适配性（1-10分，权重15%）**
- 9：匹配 社会/民生/工资/失业/裁员/教育/房价/物价/养老/医保/公平/执法/食品安全/国际政治/外交
- 7：匹配 商业/财经/消费/汽车/军事
- 5：匹配 海外/科技/AI/机器人/数码
- 2-3：匹配 明星/娱乐/饭圈/游戏/电竞

**天然分裂度（1-10分，权重10%）**
- 9：匹配 应不应该/对不对/合理/公平/能不能/该不该/值不值/值得吗
- 8：匹配 争议/不同意见/看法/你怎么/两派/网民争论
- 6：匹配 降价/涨价/下跌/上涨/暴跌/暴增
- 3-4：匹配 赛事/比赛/冠军/获胜/进球/世界杯（无对立面）
- 5：以上皆无（中位数）

### 交叉验证加分

同一话题在≥2个平台同时出现，总分额外+0.5分。检查方法：取话题标题前8-10字，逐个比对各平台items的title。

## 阶段2：去重检查

先列出所有历史话题的标题，逐条比较候选话题的语义角度，而非机械比较关键词。

### 两步去重法

```
Step 1：关键词快速过滤（自动）
  扫描候选标题中的词 vs 历史标题中的词
  如果≥3个词匹配 → 标记为"疑似重复"需要深入检查
  如果1-2个词匹配（如"黄金""汽车"这种常见词）→ 保持通过，标记为"低风险"

Step 2：语义角度判断（人工，Agent自己做）
  对"疑似重复"话题问：
  - 核心角度是否与历史话题相同？
    "金价跌了" vs "央行买金但金价还是跌了" → 角度不同，不算重复
    "AI替代程序员" vs "AI让程序员失业" → 角度相同，算重复
  - 时间：间隔>4天且角度全新 → 可用。间隔<4天 → 谨慎，一般为重复。
```

**关键原则**：宁可多选一个角度相似的话题（后期可以换），不要因为关键词false-positive漏掉顶级话题。

## 阶段3：手动调优（最高分不一定最爆）

评分只是过滤器，不是选择器。最终5个话题需要人工判断：

1. **多样性检查**：5个话题应覆盖消费/教育/职场/社会/国际不同领域，连续2个同一子类的话题最多不超过2个。
2. **钱包距离优先**：评分差距在0.5分以内时，选钱包距离更近的那个。
3. **翻转角度检查**：每个话题是否找到了"翻转观点"？如果只能做表面描述，降权。
4. **底层社会矛盾**：每个话题背后是否对应一个真实存在的结构性矛盾？（贫富差距/教育不公/代际冲突/阶层固化）

## 阶段4：生成PPT

评分完成的topics列表写入文件，传给PPT脚本。**注意JSON格式必须为扁平list**（详见`references/pptx-topics-json-format-pitfall.md`）。

### 字段要求

每个topic dict必须包含：
```python
{
    "title": str,          # 话题标题
    "score": float,        # 总分（如8.8）
    "grade": str,          # "S" 或 "A"（⚠️ 不是"rating"，不是"grade_level"）
    "scores": {            # 5个维度分数，KEY必须为中文（脚本直接当label展示）
        "钱包距离": int,      # 权重35%，1-10分
        "反驳成本": int,      # 权重25%，1-10分
        "物件锚点": int,      # 权重15%，1-10分
        "头条适配": int,      # 权重15%，1-10分
        "天然分裂": int       # 权重10%，1-10分
    },
    "source": str,         # 数据来源平台（如"百度热搜"）
    "heat": str,           # 热度值（如"#23"或"140万热度"）
    "angle": str,          # 核心翻转角度（1-2句话）
    "object_detail": str,  # 具体物件（逗号分隔，可选）
    "split_detail": str,   # 两派是谁（各自一句话，可选）
    "reason": str          # 推荐理由（用于PPT第7页，可选，不提供时自动从angle截取）
}
```

**⚠️ 常见陷阱：**
1. `grade` 字段名 → 不是 `rating`。脚本 `draw_topic_block()` 硬编码 `topic['grade']`，用 `rating` 会报 `KeyError`。
2. `scores` 的 key 必须是**中文** → `钱包距离`/`反驳成本`/`物件锚点`/`头条适配`/`天然分裂`（与评分维度完全一致）。脚本遍历 `scores.items()` 直接将 key 作为文本标签打印到PPT上。如果用英文（如 `wallet`），PPT会显示 "wallet 8/10" 而非 "钱包距离 8/10"。
3. `object_detail` 和 `split_detail` 可选（脚本用 `.get()` 访问，缺省时显示空）。
4. `reason` 可选，缺省时脚本自动从 `angle` 前60字截取。
