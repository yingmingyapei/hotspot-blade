# Hotspot Blade v4.1 升级方案（2026-06-17）

> 来源：WSL Hermes 与 iStoreOS Hermes 讨论确定

## 优先级排序

| 优先级 | 任务 | 预期效果 | 状态 |
|--------|------|----------|------|
| P0 | simhash 去重脚本化 + 禁用语清单更新 | 消除重复话题 + 消除 AI 味 | 待执行 |
| P1 | curl_cffi 替代 curl+Cookie + 数字校验脚本 | 解决 Cookie 过期 + 防止编数据 | 待执行 |
| P2 | 分级模型策略 + cron 输出格式标准化 | cron 省 token / 手动出质量 | 待执行 |

## P0：去重脚本化

**核心原则**：去重这种机械操作不要交给 LLM，写 Python 逻辑硬匹配。

方案：
1. simhash 硬去重脚本（不走模型，0 token 0 误判）
2. history.json 记录 SHA256 哈希
3. 脚本先过滤再进入写作流程
4. 双重确认：脚本筛重复标题 + LLM 做最后的内容去重（同一事件不同角度）

## P0：禁用语清单（合并自两个 Agent）

### 禁止词
- 值得注意的是
- 在...方面
- 发挥着重要作用
- 不可或缺
- 不仅...更
- 某种意义上
- 从某种角度来看
- 众所周知
- 不可否认
- 毋庸置疑
- 业内人士指出（除非真引用）
- 被广泛认为
- 引起了广泛关注（直接说谁关注了）
- 背后反映了（文章不是论文）
- 引发热议（自己说的热点不需要自证）

### 禁止句式
- 是...的句式（甲是乙的丙 → 直接说甲做了丙）
- 被字句过多
- 排比三连
- 在当今这个...的时代（开头直接砍）

### 防止 LLM 编数据
1. prompt 里写"不确定的数据用'据报道''有数据显示'代替具体数字"
2. 写作后用脚本扫描文章中的数字（正则 `\d+%`/`\d+亿`），与原始数据源对比
3. 热点文章尽量少用数据，多用故事和观点

## P1：数据采集层

- curl_cffi 替代 curl+Cookie（impersonate 参数模拟浏览器指纹，不需要 Cookie）
- Scrapling 做页面解析（比正则/BeautifulSoup 灵活）
- 国内平台（头条/微博）优先 browser_navigate，curl_cffi 反而不行

## P2：分级模型策略

| 场景 | 模型 | 原因 |
|------|------|------|
| cron 自动执行 | MiMo | 成本低，质量凑合 |
| 用户手动触发 | DeepSeek/Claude | 出质量 |

## 写作优化经验（来自 iStoreOS Hermes）

1. **具体模板代替抽象要求** — "第一段写冲突" 而不是 "写吸引人"
2. **禁用语列表** — 写进 system prompt 的禁止词表
3. **反共识观点+具体案例** — 模型不喜欢写具体案例但读者爱看
4. **强模型写初稿** — 弱模型只做格式整理
5. **巴菲特文风配方** — 生活类比 30% + 数据 25% + 冷幽默 20% + 第一人称 15% + 金句 10%
