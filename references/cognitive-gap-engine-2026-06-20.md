# 认知落差引擎（Cognitive Gap Engine）

> 来源：2026-06-20 WSL+IA 17轮结构化辩论第16轮产出

## 问题

Russell Flip Arsenal 的翻转引擎翻的是**立场**（站A还是站B）。但读者对大多数热点话题已有预设立场，立场型翻转容易变成"为了反转而反转"——读者一眼看出是AI硬拗的。

真正让读者"拍案"的，不是翻立场，而是**颠覆默认前提**——让读者发现自己一直以为是地基的东西，其实是浮冰。

## 核心机制

在 topic-scorer 和 toutiao-viral-writing 之间插入一步认知落差检测：

1. **找默认共识**：读者对这个话题的"无需思考就接受的结论"是什么？
2. **颠覆前提**：如果这个共识的前提条件刚好是反的，会发生什么？

引擎不输出最终观点，只输出一个**"让读者先愣住0.5秒"的假设**。

## 示例

| 话题 | 默认共识 | 颠覆前提的假设 |
|------|---------|--------------|
| 年轻人躺平 | 躺平是被迫的、消极的 | 如果躺平是极其理性的市场套利行为——当劳动力回报低于资本回报时，躺平反而是最高回报策略？ |
| 日本签证涨5倍 | 涨价影响赴日旅游意愿 | 如果涨价是筛选高消费游客，反而提升旅游体验和消费总额？ |
| AI取代人类工作 | AI让人类失业 | 如果AI只是揭示了一个事实：那些工作本来就不该由人类来做？ |

## 用法

### 在 prompt 链中的位置

```
topic-scorer（选题评分）
    ↓
认知落差引擎（找默认共识 → 生成颠覆假设）← 新增
    ↓
Russell-Flip-Arsenal（选择翻转角度）
    ↓
toutiao-viral-writing（九边风格写作）
    ↓
article-polish-master（润色）
```

### 参考 Prompt

```
输入热点话题。输出两行：
1. 读者默认共识（一句，概括大众对这个话题的"无需思考就接受的结论"）
2. 颠覆前提的假设（一句，反转默认共识的前提条件）

要求：假设必须尖锐但不荒谬，让读者读完"愣0.5秒"然后觉得"对，这才合理"。
200 token以内。
```

## 与 Russell Flip 的区别

| | Russell Flip | 认知落差引擎 |
|---|-------------|------------|
| 翻什么 | 立场（赞成/反对） | 前提（你认为对的，可能是错的） |
| 典型产出 | "战争不是破坏性力量，战争是人类逃避无聊的唯一出口" | "如果战争从来不是关于杀戮，而是关于让平庸生活变得有意义？" |
| 读者反应 | "这角度有意思" | "卧槽，真是这样" |
| 命中率预估 | 10-30% | 40-60%（理论值，待实测验证） |

## 注意事项

- **数据依赖（2026-06-20 32轮辩论发现）**：引擎需要事实侧数据源（统计数据+权威来源）才能产出可信的认知落差。仅靠热搜情绪侧数据（现有hotlist-scraper提供）时，引擎退化为"模型猜测"。无事实支撑时标注为"推测性落差"，降低权重。
- 不是每个话题都适合用——需要话题有"默认前提"可翻
- 过犹不及——如果默认共识本身就很准确，不要硬翻
- 优先用于社会现象、经济行为、科技伦理类话题
- 娱乐八卦、纯粹社会新闻类话题不适用
