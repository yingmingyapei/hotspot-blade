# 字数执行教训（2026-05-31 实测 + 2026-06-11 更新）

## 问题

热点刀锋 cron job 生成的微头条平均中文字数仅 268 字，远低于 500 字目标。

| # | 话题 | 纯中文字数 |
|---|------|-----------|
| ① | 多所高校撤销外语学院 | 281 |
| ② | 胖东来矿泉水停售 | 275 |
| ③ | 蛋白粉价格暴涨 | 271 |
| ④ | 日本没资格谈防务 | 234 |
| ⑤ | 空气炸锅纸致癌 | 280 |
| **平均** | | **268字** |

## 根因

模板中验证阈值设为 `chinese < 300`，模型将 300 当作上限而非底线。
"500字左右"的表述太软，没有硬性约束力。

## 修复（已应用）

1. **验证阈值**：`chinese < 300` → `chinese < 400`
2. **字数要求措辞**：从"500字左右"改为"底线400字，目标500字，上限600字"
3. **写法指导**：新增结构化分配——开场100字+中段300字+结尾100字
4. **cron job prompt**：在最顶部加三重⚠️字数硬性要求

## 教训

- **验证阈值必须≥目标的80%**：目标500字，验证阈值至少400字
- **"左右"是无效约束**：模型会贴着下限跑，必须用"底线X字"明确下限
- **字数要求放在 prompt 最顶部**：放在中间容易被忽略
- **写法指导要具体化**：告诉模型每个段落写多少字、怎么展开

## 补充教训（2026-06-02 实测）

### 初稿低于400字的最常见根因

中段"展开论证"写得太薄——把本该展开2-3句的论点压缩成一句话带过。

**实测案例**（Article 1 初稿仅354字）：
```
✗ 初稿写法（一句话带过，约15字）：
  "有人会说才卖32枚你就说崩？确实是九牛一毛。但碎掉的不是钱，是那个'永远'。"

✓ 修改后（展开2-3句+类比，约50字）：
  "有人会说才卖32枚你就说崩？确实是九牛一毛。但碎掉的不是钱，是那个'永远'。
   就像教堂里的牧师开始偷偷卖圣经，数量不多，但动摇的是信仰本身。"
```

### 四种展开技巧（选一种即可）

1. **类比展开**：用生活化比喻把抽象论点具象化（"就像……"）
2. **数据展开**：补充一个具体数字/案例/场景
3. **对话展开**：引用一方观点，再用事实反驳
4. **场景展开**：描写一个具体画面让读者代入

### 预判公式

中段3个论点 × 每个展开后约50字 = 中段150字。加上开场100字 + 结尾80字 = 330字。
要达到400字：中段需要3-4个论点，或每个论点展开到60-70字。
写完后如果不足，优先给最薄弱的论点补展开，而不是水词填充。

## 后写扩展模式（2026-06-11 实测沉淀）

当初稿完成后 word count 仍低于目标（实测：初稿 300-486 字，目标 500 字），使用以下系统化扩展技巧，每轮增加约 50-100 字：

### 扩展优先级

1. **补具体人名/品牌名**：`车主` → `车主赵女士`，`客服` → `小米客服`
2. **补具体数字**：`医药费` → `骨折医药费106块9`，`误工费` → `恢复期一到两个月`
3. **补场景展开**：增加一个让读者代入的具体画面（停车场拎菜抱孩子关门的场景）
4. **补因果链条**：A → B → C 的逻辑链中间环节（美联储不降息→美元强→人民币承压→基金缩水→房贷月供不降）
5. **补对比/类比**：美国通胀高 vs 中国通胀低的对比结构

### 扩展禁区

- ❌ 不用水词填充（"众所周知""不可否认"等）
- ❌ 不用重复论述（换个说法重说同一个观点）
- ❌ 不用空泛总结（"总之""综上"）

### 扩展流程

```
Step 1: 写完初稿 → python3 检查中文字数
Step 2: 如果 < 目标字数 → 识别最薄弱的段落（通常是最短的那个）
Step 3: 从上述5种技巧中选1-2种扩展该段落
Step 4: 重新检查字数 → 如果仍不足，扩展第二薄弱段落
Step 5: 达标后做退役句式扫描
```

### 实测数据（2026-06-11）

| 话题 | 初稿字数 | 扩展后字数 | 扩展手法 |
|------|---------|-----------|---------|
| 小米SU7电吸门 | 401 | 501 | 补人名+补数字+补场景+补因果 |
| 美国CPI | 305 | 399 | 补因果链条+补对比结构+补具体场景 |
| 金价暴跌 | 347 | 419 | 补具体人名+补数字+补类比扩展 |

### 与前写扩展的区别

| 维度 | 前写扩展（写之前） | 后写扩展（写之后） |
|------|-------------------|-------------------|
| 时机 | 选题确认后、动笔前 | 初稿完成后 |
| 方法 | 结构化分配字数（开场100+中段300+结尾100） | 识别最薄弱段落，用5种技巧扩展 |
| 目标 | 预防字数不足 | 补救字数不足 |
| 效果 | 初稿即达标（理想情况） | 1-2轮扩展后达标（兜底方案） |

**最佳实践**：前写扩展为主，后写扩展为兜底。两层保障确保字数达标。
