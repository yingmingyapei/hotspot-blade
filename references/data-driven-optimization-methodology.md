## 核心发现

### 点击率是第一指标

头条算法核心公式：
```
推荐量 = 初始池（500-1000人）× 数据系数
数据系数 = 点击率 × 读完率 × 互动率 × 停留时长
```

**关键阈值**：
- 点击率 < 5% → 不晋级，算法停止扩大推荐
- 点击率 > 10% → 重点放大推荐
- 互动率 > 3% → 重点放大推荐

## 头条适配性评分机制

**核心问题**：热榜话题≠头条用户兴趣，必须优先筛选头条用户真正关心的话题。

**评分标准（1-10分）**：

| 维度 | 分值 | 说明 |
|------|------|------|
| 有矛盾/靶子 | +3分 | 九边风格核心 |
| 涉及普通人利益 | +2分 | 不是精英/极客专属 |
| 有争议空间 | +2分 | 评论区能形成两派 |
| 有具体数字/案例 | +1分 | 不是空泛讨论 |
| 贴近中国社会现实 | +1分 | 不是海外/科技专属 |
| 领域加分 | +1分 | 国际政治/社会民生 |
| 领域减分 | -1分 | 科技/AI |

**分级处理**：
- 8-10分：S级，优先选择
- 6-7分：A级，可选
- 4-5分：B级，降级处理
- <4分：C级，直接排除

## 科技类内容降维检查清单

当候选话题命中科技/AI/编程等关键词时，必须通过以下检查：

**标题降维**：
- 标题是否包含专业术语？→ 必须替换为通俗表达
- 标题是否突出"对普通人的影响"？→ 必须包含"你的XX"
- 标题是否有具体数字？→ 必须有数字冲击

**正文降维**：
- 第一段是否用生活化类比？
- 是否避免了技术细节？
- 是否用"你每天用的XX"来类比？

**结尾降维**：
- 追问是否具体到普通人有话说？
  - ✅ "你用过AI写代码吗？体验怎么样？"
  - ❌ "你怎么看AI编程的未来？"

**如果无法降维，直接放弃该话题！**

## 结尾追问模板库

### 高互动率追问（实测有效）

| 追问类型 | 模板 | 适用场景 |
|---------|------|---------|
| 具体数字追问 | "你家那边呢？涨了多少？" | 物价/涨价类 |
| 二选一追问 | "你猜是A赢了，还是B赢了？" | 对立观点类 |
| 体验追问 | "你用过XX吗？体验怎么样？" | 科技/产品类 |
| 身边故事追问 | "你身边有这样的人吗？" | 社会现象类 |
| 选择追问 | "如果是你，你会怎么选？" | 道德困境类 |

### 避免使用

- "你怎么看？"（太泛）
- "评论区聊聊"（无具体方向）
