## 优化决策

### 1. 数据源权重调整

| 数据源 | 旧权重 | 新权重 | 理由 |
|--------|--------|--------|------|
| 百度热搜 | 25% | 30% | 国内大众热点，最接近头条用户 |
| 头条站内 | 0% | 25% | 直接命中头条用户兴趣（新增） |
| 知乎热榜 | 0% | 20% | 深度社会议题，九边风格最佳素材（新增） |

| YouMind | 25% | 5% | 海外爆款追踪，偶尔有用 |
| HN Firebase | 25% | 已弃用 | 科技向，头条用户不买账 |

### 2. 头条适配性评分（新增核心筛选器）

评分标准（1-10分）：
- 有矛盾/靶子 +3分
- 涉及普通人利益 +2分
- 有争议空间 +2分
- 有具体数字/案例 +1分
- 贴近中国社会现实 +1分
- 领域加分：国际政治+1，社会民生+1，科技-1

只保留>6分的话题。

### 3. 选题权重配置

- 国际政治40%（实测10-13%点击率）
- 社会民生25%（实测5-10%）
- 军事15%
- 商业财经10%
- 科技5%（必须降维）
- 海外信息差5%

硬约束：每天5篇中国际政治+社会民生必须占3篇以上。

### 4. 知乎/微博抓取方案

定时任务环境下opencli的API命令（zhihu hot/weibo hot）需要Extension连接，headless环境会超时。

解决方案：百度搜索间接获取
```bash
python3 -c "
import urllib.request, re
url = 'https://www.baidu.com/s?wd=知乎热榜+今日+热门话题'
req = urllib.request.Request(url, headers={'User-Agent': 'Mozilla/5.0'})
resp = urllib.request.urlopen(req, timeout=15)
text = resp.read().decode('utf-8')
titles = re.findall(r'<h3[^>]*>(.*?)</h3>', text, re.DOTALL)
..."
```

## 完整优化闭环

```
每天8:00 热点刀锋执行
    ↓ 使用当前选题权重+标题公式
    ↓ 生成5篇微头条
    ↓ 积累数据
每周一12:00 toutiao-weekly执行
    ↓ 分析上周数据（点击率/互动率/撕裂指数）
    ↓ 自动调整选题权重
    ↓ 更新标题公式白名单
    ↓ 下周热点刀锋使用新权重
```
