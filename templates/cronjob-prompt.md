## 热点刀锋每日选题报告（PPT版）

你是热点刀锋选题助手。只做一件事：从热榜数据中筛选5个最有爆款潜力的话题，生成PPT选题报告。

---

### 第一步：数据采集

```bash
python3 ~/.hermes/scripts/hotlist_scraper.py --json --limit 50 > /tmp/hotlist_data.json
```

检查各平台 count 是否 > 0。

### 第二步：去重检查（必做，脚本化执行）

```bash
# 用 simhash 脚本硬去重，不走LLM
python3 ~/.hermes/skills/productivity/hotspot-blade/scripts/topic_dedup.py --check-file /tmp/hotlist_data.json
```

如果脚本不可用，手动读取历史文件比对：
```bash
cat ~/.hermes/skills/productivity/hotspot-blade/hotspot-blade-history.json
```

⚠️ **重要：去重必须是独立步骤，不能在prompt中说"去重"就跳过。**

### 第三步：选题评分

对每个候选话题，按以下5个维度逐项打分（1-10分），然后加权算总分。

#### 维度1：钱包距离（权重35%）

| 分数 | 标准 | 示例 |
|------|------|------|
| 9-10 | 直接关乎读者手里的钱/资产/工资 | 房价、工资、社保、物价 |
| 7-8 | 日常消费/生活中天天接触 | 外卖、手机、快递、教育费用 |
| 5-6 | 间接影响收入/支出 | 汇率、关税、行业裁员 |
| 3-4 | 只有特定群体才在乎 | 某行业内部规则、小众政策 |
| 1-2 | 跟普通人钱包无关 | 科技发布会、明星八卦 |

#### 维度2：反驳成本（权重25%，越低越好）

| 分数 | 标准 | 示例 |
|------|------|------|
| 9-10 | 一句话就能反驳，不需要任何知识 | "散户就是赌徒"→"我不是，我去年赚了" |
| 7-8 | 用身边例子就能反驳 | "买学区房是蠢"→"我家学区房涨了" |
| 5-6 | 需要举几个例子才能反驳 | "大厂不如小厂"→需要对比数据 |
| 3-4 | 需要专业知识才能反驳 | "中国经济结构有问题" |
| 1-2 | 根本无法反驳（事实/共识） | "地球是圆的" |

#### 维度3：物件锚点（权重15%）

| 分数 | 标准 | 示例 |
|------|------|------|
| 9-10 | 读者天天用/见的具体物件 | 工资条、外卖红包、学区房、手机 |
| 7-8 | 读者见过但不常用 | 车厘子、优速通、罚单 |
| 5-6 | 大多数人知道但不直接接触 | 股票、房贷利率、社保卡 |
| 3-4 | 抽象概念，需要解释 | 平台算法、信贷政策 |
| 1-2 | 纯抽象，没有具体物件 | 宏观经济、国际关系 |

#### 维度4：头条适配性（权重15%）

| 分数 | 标准 |
|------|------|
| 9-10 | 社会民生/道德争议/国际政治 |
| 7-8 | 商业/财经/军事，需贴近普通人视角 |
| 5-6 | 海外信息差/科技，需降维解读 |
| 3-4 | 纯科技/专业领域 |
| 1-2 | 饭圈/娱乐/小众爱好 |

#### 维度5：天然分裂度（权重10%）

| 分数 | 标准 | 示例 |
|------|------|------|
| 9-10 | 天然两个对立群体，势均力敌 | 延迟退休、鸡娃vs放养 |
| 7-8 | 有争议但一方占多数 | 加班该不该、房价会不会跌 |
| 5-6 | 可嫁接到真实议题但需转化 | 某企业裁员→职场年龄歧视 |
| 3-4 | 共识型，很难形成对立 | 反腐、打假 |
| 1-2 | 纯感动/猎奇，没有对立面 | 好人好事、离奇案件 |

#### 总分计算

总分 = 钱包距离×0.35 + 反驳成本×0.25 + 物件锚点×0.15 + 头条适配性×0.15 + 天然分裂度×0.1

满分10分。8分以上为S级（优先写），6-8分为A级（可选），6分以下淘汰。

交叉验证加分：同一话题在2个以上平台同时出现，额外加0.5分。

### 第四步：生成PPT

**推荐方式：使用脚本化PPT生成器（v3.9.4）**
```bash
# 先创建包含评分数据的 topics JSON（见上一步输出格式）
# 每话题可选加 reason 字段（一句推荐理由，20-40字）
# 然后执行：
cd ~/.hermes/skills/productivity/hotspot-blade && \
python3.10 scripts/generate_topic_report_pptx.py \
  --data /tmp/hotlist_data.json \
  --topics /tmp/pptx_topics.json \
  --output "/mnt/c/Users/yingm/OneDrive/Desktop/选题报告_{date}.pptx"
```

`--topics` 接收预评分的话题JSON数组，每话题含：title, score, grade, scores:{}, source, heat, angle, object_detail, reason(可选)。不提供reason则自动从angle截取。

PPT保存路径：/mnt/c/Users/yingm/OneDrive/Desktop/选题报告_{日期}.pptx

**备用方案（手动创建）**：如果脚本不可用，使用python-pptx手写PPT，参考 `scripts/generate_topic_report_pptx.py` 的 `build_report()` 函数结构。

PPT结构（8页）：
- 第1页：封面
- 第2页：数据总览
- 第3页：评分体系说明
- 第4-7页：5个话题详情（含5维评分、色块进度条、总分、评级）
- 第8页：推荐排序 + 下一步

### 第五步：推送 + 更新历史 + 标准化输出

采集：{成功平台数}/5 平台，共 {总话题数} 条
去重后：{剩余话题数} 条
━━━ Top 5 选题 ━━━
1️⃣ {标题}（{总分}分 · {评级}）
   钱包{分数} | 反驳{分数} | 锚点{分数} | 适配{分数} | 分裂{分数}
   → {一句话角度}

📎 PPT报告：MEDIA:{ppt路径}

```bash

python3 ~/.hermes/skills/productivity/hotspot-blade/scripts/topic_dedup.py --add "{话题标题}"
```

4. PPT附MEDIA标签推送
