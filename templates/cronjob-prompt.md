## 热点刀锋每日执行（v3.5 curl+Cookie版）

执行「热点刀锋」(hotspot-blade) 技能，全流程自动执行。

---

## 数据采集方式：curl + Cookie + Python

**所有数据采集通过热榜抓取脚本完成：**

```bash
python3 ~/.hermes/scripts/hotlist_scraper.py --json --limit 50
```

该脚本使用 curl + Cookie 直接调用各平台 API，无需 opencli、无需 Chrome、无需浏览器登录态。

- Cookie 文件：`~/.hermes/cookies/hotlist-cookies.json`
- 支持平台：知乎热榜、微博热搜、B站热门、头条热榜、百度热搜
- **脚本返回 JSON 格式，直接解析即可**
- **如脚本报错，检查 Cookie 是否过期（知乎z_c0有效期约5个月，B站SESSDATA约6个月）**

---

## 执行步骤

### 第一步：数据采集

```bash
python3 ~/.hermes/scripts/hotlist_scraper.py --json --limit 50 > /tmp/hotlist_data.json
```

一条命令抓取全部5个平台。检查输出中各平台 count 是否 > 0。

### 第二步：数据分析（v3.4选题权重）

通读所有热榜原始数据，按以下权重筛选话题：

| 维度 | 权重 | 说明 |
|------|------|------|
| 阶层/消费关联度 | 30% | 59%爆款来自这两个主题，优先选 |
| 反驳成本（越低越好） | 25% | 读者能一句话反驳的话题评论最多 |
| 具体物件锚点 | 15% | 有具体物件的话题评论数平均高3倍 |
| 头条适配性 | 25% | 是否命中头条用户兴趣 |
| 议题分裂度 | 15% | 能否嫁接到真实议题形成对撞 |

**物件锚点检查**：每个候选话题必须有一个读者生活中见过的具体"东西"。没有具体物件的话题降权。

**外国映射加分**：可以讲外国故事的话题 +15分。

- 识别社会矛盾和撕裂点
- 交叉验证多平台热点
- 排除纯娱乐/饭圈/无争议新闻
- 去重（排除7天内已写话题）
- **选定2个核心话题**

### 第三步：微头条生成（v3.4一题三篇模式）

2个核心话题 × 每个话题2-3篇不同角度：
- **第一篇：引爆** — 抛出笃定观点，引爆评论区
- **第二篇：反驳** — 引用评论区反对声音，针对性反驳
- **第三篇：升华** — 拉到更高维度，收束话题

**v3.4强制规则**：
- 每篇≥400中文字，目标500字
- **自曝式开场**：前30字必须包含"我"字
- **物件锚点**：每篇必须有一个具体物件
- 加载 article-polish-master 做终稿润色

**⚠️ 一题三篇数据弹药隔离规则（必做）**：
- 同一数据点（具体数字、公司名、事件细节）**最多出现在1篇中**，不跨篇复用
- 第1篇（引爆）：宏观数据+大数字冲击
- 第2篇（反驳）：具体案例+个人故事
- 第3篇（升华）：类比+历史对比+哲学视角
- **自检**：写完后逐篇扫描，发现任何数据点重复出现在2篇以上 → 必须替换为新数据

### 第四步：推送 + 存档
- 使用 `hermes send -t "telegram:6327421932"` 分条推送（每篇一条消息，避免截断）
- 存档到 `/mnt/c/Users/yingm/wiki/sources/market-intelligence/daily/{date}-热点刀锋微头条-5篇.md`
