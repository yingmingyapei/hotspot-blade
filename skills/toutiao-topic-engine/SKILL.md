# 选题引擎与微头条公式

> 爆款刀锋的选题与短内容模块。选题决定生死，钩子决定点击。

---

## 一、营销驱动选题法

### 核心理念

选题不是选最热的话题，是选离用户钱包最近的话题。

头条用户要"体感"："跟我有什么关系？我的钱会不会变少？"

### 选题距离评估表

| 话题类型 | 与用户距离 | "我的钱"关联度 | 能否一句话反驳 |
|----------|-----------|---------------|--------------|
| 房产/资产 | 零距离 | ★★★★★ | ✅ |
| 手机/消费电子 | 近 | ★★★★☆ | ✅ |
| 外卖/物价 | 近 | ★★★☆☆ | ✅ |
| 金价/理财 | 中 | ★★★★☆ | ✅ |
| AI/程序员 | 远 | ★★☆☆☆ | ✅ |
| 医疗/行业规则 | 远 | ★★☆☆☆ | ❌ |
| 汇率/宏观经济 | 太远 | ★☆☆☆☆ | ❌ |

### 选题优先级公式

```
选题价值 = 物件锚点 × 反驳成本 × 钱包距离
```

- 热度高但离钱包远 = 旁观看热闹，评论区死水
- 热度中但离钱包近 = "这跟我有关"，评论区炸锅

## 二、五维选题评分体系

| 维度 | 权重 | 说明 |
|------|------|------|
| 钱包距离 | 35% | 跟读者的钱有没有关系 |
| 反驳成本 | 25% | 越低越好，一句话就能反驳 |
| 物件锚点 | 15% | 读者生活中见过的东西 |
| 头条适配 | 15% | 适合头条算法推荐 |
| 天然分裂 | 10% | 天然有正反两派 |

**核心洞察**：钱包距离 + 反驳成本 = 情绪操控的双引擎。

## 三、人性密码五公式

> 爆款和文笔无关，只和"人性套路"有关。占贪嗔痴慢疑一样，想不爆都难。

### 公式一：情绪钩子（开局炸裂）

```
模板：[人物]把[物品]给了[对立]，[主角]当场[动作]，[另一方]却说我[评价]。
示例：婆婆把我女儿的奶粉偷偷喂了她外孙，我当场翻脸，老公却说我小题大做。
适用：家庭矛盾、职场冲突、社会不公
```

### 公式二：干货清单（数字+反差）

```
模板：[年龄]后，我逼自己养成的[数字]个"[反差词]"习惯，越[动作]越[结果]。
示例：30岁后，我逼自己养成的5个"烧钱"习惯，越花越有钱。
适用：省钱技巧、经验总结、避坑指南
```

### 公式三：故事反转（悬念留白）

```
模板：[人物][事件]，我[动作]，[时间后][转折事件]，[结果]……
示例：闺蜜嫁了富二代，我随礼2000，半年后她深夜哭着敲开门，一开口我傻了……
关键："展开全文"的欲望 = 推荐量的密码。
```

### 公式四：热点借势（态度即流量）

```
模板：[热点事件]，最可怕不是[表层]，而是[深层]。
示例：李某某事件，最可怕不是塌房，而是年轻人价值观的扭曲。
红线：不碰政治敏感 / 不碰未证实谣言 / 不碰平台禁推话题
```

### 公式五：互动诱导（吵起来就赢了）

```
模板：如果[假设情境]，你愿意[选择]吗？评论区说出你的理由，我每条都看。
示例：如果老公年薪百万，你愿意当全职妈妈吗？
效果：评论越多，系统推荐越猛。
```

### 人性密码选题对照表

| 人性密码 | 选题方向 | 适用公式 |
|----------|---------|---------|
| 贪（占便宜） | 省钱/薅羊毛 | 干货清单 |
| 嗔（生气） | 社会不公/职场委屈 | 情绪钩子 |
| 痴（好奇） | 悬念/反转 | 故事反转 |
| 慢（炫耀） | 成功/跃迁 | 干货清单 |
| 疑（恐慌） | 风险/避坑 | 热点借势 |

## 四、微头条钩子模板库

| 钩子类型 | 模板 | 适用场景 |
|----------|------|---------|
| 情绪爆破型 | [人物]把[物品]给了[对立]… | 家庭/职场 |
| 数字反差型 | [数字]个[反差词]习惯… | 干货/经验 |
| 悬念留白型 | [铺垫]，[转折]，[未说完]… | 故事/八卦 |
| 热点态度型 | [热点]，最可怕不是… | 社会热点 |
| 互动争议型 | 如果[假设]，你愿意…？ | 价值观 |

## 五、微头条发布规范

### 字数与配图

- 字数：200-400字（超600字推荐量下降）
- 配图：情绪类1张 / 故事类3张 / 干货类9张
- 标签：1-3个精准标签

### 最佳发布时间

- 🔥 晚8-10点（睡前高峰）
- ✅ 早7-9点（通勤高峰）
- ✅ 午12-1点（午休高峰）
- ⚠️ 上午10-12点、下午2-4点表现一般

### 微头条 vs 头条文章

| 维度 | 微头条 | 头条文章 |
|------|--------|---------|
| 字数 | 200-400字 | 800-1200字 |
| 结构 | 五公式速成 | 七步工作流 |
| 核心 | 人性密码 | 九边犀利风格 |
| 测试成本 | 低（15分钟） | 高（1-2小时） |

**混合策略**：先用微头条测试选题（1-2天），数据好则扩展为头条文章。

## 六、开头随机化规则

每篇必须使用不同开头方式，禁止固定格式：

- 问句+数据开场
- 引用金句开场
- 场景描写开场
- 直陈事实开场
- 短句节奏开场
- 个人经历开场
- 痛点反问开场

❌ 禁止：每篇都用同一公式 / 每篇都用同一开场

---

## 七、结尾随机化规则

每篇必须使用不同收尾方式：

- **递进+预言**：总结+余味+反问
- **具体追问**：数字+追问句（"你家那边呢？"）
- **冷幽默收尾**：[现象]默默[动作]
- **数字留白**：数字排比+一句话定性
- **你猜收尾**：你猜X？A？还是B？

❌ 禁止：每篇都用"但更关键的是"收尾

---

## Pitfalls

### 1. 科技类内容互动率极低
粉丝画像不匹配。科技类每天最多1篇，必须"降维"。连续3天CTR<1%则暂停7天。

### 2. 标题长度失控
少于20字信息量不足，超过30字被折叠。自动检查字数，不符则重生成。

### 3. 微头条字数超限
超600字推荐量下降。严格控制200-400字。
