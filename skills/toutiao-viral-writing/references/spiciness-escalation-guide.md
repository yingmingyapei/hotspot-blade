# 加辣升级路径（Spiciness Escalation Guide）

> 来源：2026-06-08 国行iPhone微头条 + 2026-06-09 高层住宅微头条实战 + 2026-06-14 日本炸鸡66年老油。
> 用户两次反馈"辣度不够"→第三次满意。高层住宅文章用户要求"辣度加辣+观点新颖+耳目一新"→一次通过。
> 核心发现：用户要的"辣"不是更多脏话或更极端的观点，而是**从"分析问题"升级到"攻击读者的生活选择"**。
> 新发现：辣度和新颖度是两个独立维度。辣度=攻击力度，新颖度=认知刷新。

## 四维强度模型

| 维度 | 🌶️ 微辣（标准） | 🌶️🌶️ 中辣 | 🌶️🌶️🌶️ 特辣 |
|------|----------------|------------|--------------|
| **辣度**（攻击力度） | 描述现象+温和结论 | 定性靶子+笃定判断 | 身份重定义+攻击生活选择 |
| **新颖度**（认知刷新） | 常规分析 | 换角度看问题 | 翻转因果链，制造"原来如此"时刻 |
| **数据密度** | 2-3个数据点 | 4-5个数据点+个人场景 | 6+数据点+算到读者个人头上 |
| **长度** | 200-400字 | 500-700字 | 800-1200字 |

## 从微辣到特辣的升级路径

### 第一步：找到"身份重定义"的靶心

不是攻击某个现象，而是**重新定义读者的身份标签**。

| 微辣写法（描述现象） | 特辣写法（身份重定义） |
|---------------------|---------------------|
| "苹果靠生态锁住用户" | "你不是苹果的用户，你是苹果的资产" |
| "高层住宅有隐患" | "你不是房东，你是这座没法拆的建筑的终身股东" |
| "外卖有大数据杀熟" | "你不是消费者，你是算法的实验品" |
| "金价跌了" | "你不是投资者，你是在给机构送流动性" |
| "日本66年老油" | "你不是食客，你是那口老油里炸出来的一块鸡排" |

**公式：你不是[正面标签]，你是[负面标签]。**

### 第二步：从"告诉你事实"升级到"替你算一笔账"

不要只给宏观数据，要**算到读者个人头上**。

| 微辣写法 | 特辣写法 |
|---------|---------|
| "iPhone国行比美版贵2700" | "你花一万块，三千五给了中国工人，四千五揣进了库克的口袋" |
| "电梯更换费用高" | "你那套32层，300万，贷款30年，总利息146万，实际付出446万" |
| "日料很贵" | "你花300块吃日料，花30块吃兰州拉面，你花了十倍的钱买了一个别人编给你的故事" |

**公式：把宏观数据 × 读者个人场景 = 具体到让人心痛的数字。**

### 第三步：找到翻转因果链的新角度（新颖度）

不是找新话题，而是**用新因果链解释老话题**。

| 话题 | 老生常谈 | 耳目一新 |
|------|---------|---------|
| 高层住宅 | "电梯老化、消防隐患" | "6层能拆，32层永远不会拆——你买的不是家，是终身股东" |
| iPhone贵 | "苹果税、品牌溢价" | "你不是用户，你是资产——95%中国造，85%利润归苹果" |
| 日本66年老油 | "日本双标" | "区别不在油里，在谁来讲这个故事——你消费的不是食品，是叙事" |

**公式：用一个读者从没想过的因果关系，重新解释一个他们以为自己懂的话题。**

**角度升级经验**：当用户提出一个角度时，思考是否有更高一层的框架。"民族情绪"→"叙事权"就是从"发泄"升级到"揭示机制"，用户通常接受更好的方案。

### 第四步：结尾追问要"扎到肉里"

不要问"你怎么看"，要问一个**让读者忍不住去查、去算、去反驳**的具体问题。

| 弱追问 | 强追问 |
|--------|--------|
| "你怎么看高层住宅？" | "你住几层？你算过你那栋楼的电梯维修基金还剩多少吗？" |
| "你用iPhone吗？" | "你每个月给苹果交多少保护费？" |
| "你觉得日本食品怎样？" | "你愿意为哪个版本的故事买单？你不是食客，你是那口老油里炸出来的一块鸡排" |

**公式：把抽象话题变成一个读者可以立刻行动（查、算、问）的具体动作。**

## 用户反馈信号→调整动作映射

| 用户说的 | 实际意思 | 调整动作 |
|---------|---------|---------|
| "辣度不太够" | 还在描述现象，没攻击选择 | 升级到身份重定义 + 攻击读者的生活选择 |
| "辣度要再加" | 身份重定义已有，但还没攻击到读者具体行为 | 从"认知层面"升级到"消费行为层面" |
| "还要再加辣" | 攻击行为已有，但收尾冲击力不够 | 最终身份重定义 + 一句扎心的比喻收尾 |
| "观点不够吸引人" | 论点太常规，读者见过 | 找一个翻转因果链的新角度 |
| "看完之后有种让人耳目一新的感觉" | 需要新颖度 | 用新因果链解释老话题，制造"原来如此"时刻 |
| "好，过瘾" | 当前版本合格 | 不要再改了，保持这个力度 |
| "可以，直接发Telegram" | 用户满意，直接交付 | send_message 到 Telegram |

## 实战案例

### 案例1：国行iPhone（2026-06-08）

- 初版（微辣）："苹果靠生态锁住用户" → 用户："辣度不够"
- 终版（特辣）："你不是用户，你是资产" + "95%中国造，85%利润归苹果" + "你每个月给苹果交多少保护费？" → 用户："好，过瘾"

### 案例2：高层住宅（2026-06-09）

- 用户要求："辣度加辣，观点新颖，耳目一新"
- 一次通过版本："6层能拆，32层永远不会拆" + "你不是房东，你是终身股东" + "你住几层？电梯维修基金还剩多少？"
- 关键新颖点：从"高层有隐患"（老生常谈）翻转为"永远不拆=无法退出的终身股东"（耳目一新）

### 案例3：日本炸鸡66年老油（2026-06-14）— 叙事权角度 + 三轮升级

- 用户最初建议"民族情绪"角度，提出"叙事权"角度更高一层（不是发泄情绪，是揭示叙事机制），用户接受升级
- 初版（中辣）：描述双标现象+引用日本食品安全事故数据 → 用户："辣度要再加"
- 二版（特辣）：加入"你已经被那锅油炸熟了"+ 自动程序隐喻（看到日本老人→匠人精神；看到中国老人→碰瓷） → 用户："还要再加辣"
- 终版（特辣+）：攻击读者具体消费行为（花300吃日料发朋友圈配文"匠心"）+ 最终身份重定义"你是那口老油里炸出来的一块鸡排，你被炸透了还以为自己是金黄色的" → 用户："可以，直接发Telegram"
- **辣度升级路径**：攻击目标从"现象"→"读者的认知"→"读者的具体消费行为"逐级逼近
- **角度升级经验**：当用户提出一个角度时，思考是否有更高一层的框架。用户通常接受更好的方案

## 加辣自检清单

写完初稿后，逐项检查：

```
【辣度检查】
□ 有没有一句话是"你不是X，你而是Y"式的身份重定义？
□ 有没有把宏观数据算到读者个人头上（"你那套房子""你每个月"）？
□ 结尾追问是否让读者忍不住去查/算/反驳？
□ 全文读完，读者会不会觉得"这篇文章就是在说我"？
□ 如果把文章里的"你"换成"有些人"，力度是否下降？（如果下降，说明"你"用对了）

【新颖度检查】
□ 有没有一个读者从没想过的因果关系？
□ 读完第一段，读者会不会想"我怎么没想到"？
□ 这篇文章的核心论点，读者在其他地方见过吗？（如果搜到类似观点，换角度）

【交付检查】
□ 用户要求"加辣"→特辣版本（800-1200字）
□ 用户要求"控制长度"→标准版本（200-400字）
□ 用户没提要求→中辣版本（500-700字）作为默认
□ 满意后直接交付，不再追问"是否满意"
```

## 注意事项

- "加辣"不等于"加脏话"或"加极端词"。辣度来自**认知冲击**，不是情绪宣泄。
- 身份重定义必须有数据支撑，否则就是空喊口号。
- 攻击读者的选择，不攻击读者的人格。"你的选择很蠢"可以，"你很蠢"不行。
- 每次用户说"辣度不够"，先检查是不是还在"描述现象"阶段，没有升级到"攻击选择"阶段。
- "新颖度"和"辣度"是两个独立维度，可以独立调节。最理想是两者同时拉满。
- 用户说"好，过瘾"后不要再改了——这是正向反馈，保持当前力度。
- 当用户提出一个角度时，先思考有没有更高一层的框架再动笔。不要直接用用户的原始角度。
