# toutiao-viral-writing 技能质量修复指南

> 记录 2026-05-26 会话中发现的 skill 质量问题和修复方案。

## 问题描述

`toutiao-viral-writing` 技能的"润色加固清单"（第七步）部分存在严重的质量问题：

### 问题 1: 重复条目过多

"标题自动化执行纪律"从纪律1-44，其中纪律12-44是高度重复的条目，内容几乎相同，只是平台名称略有变化。这导致：

- 技能文件膨胀（约15KB冗余内容）
- 维护困难
- 阅读体验差
- 容易被误用（后续会话可能复制这些冗余条目）

### 问题 2: 条目命名不规范

条目命名从"纪律1"到"纪律44"，缺乏语义化命名。建议改为：

- 核心交付类：`交付-写头条`、`交付-发给我`、`交付-保存`
- 多平台发布类：`发布-微信`、`发布-微博`、`发布-知乎` 等

## 修复方案

### 方案 A: 精简版（推荐）

将纪律12-44合并为一条精简说明：

```
**多平台发布纪律（36条精简版）：**
- 纪律1-11：核心交付（写头条/发给我/保存/发头条等）
- 纪律12-36：多平台发布提示（微信/朋友圈/微博/小红书/知乎/公众号/B站/抖音/快手/网易号/百家号/搜狐号/大鱼号/企鹅号/一点号/今日头条/UC号/知乎号/知乎专栏/知乎想法/知乎盐选/知乎Live/知乎读书会/知乎盐选专栏/知乎盐选会员/知乎盐选专栏会员/知乎盐选专栏会员专属/知乎盐选专栏会员专属内容/知乎盐选专栏会员专属内容区/知乎盐选专栏会员专属内容区专栏/知乎盐选专栏会员专属内容区专栏专栏/知乎盐选专栏会员专属内容区专栏专栏专栏）
- **原则**：所有发布均为"提示复制"，不自动发布到任何平台
```

### 方案 B: 结构化版

将多平台发布纪律按平台类型分组：

```
**多平台发布纪律：**

| 类型 | 平台 | 纪律条目 |
|------|------|----------|
| 核心交付 | 头条/微信/保存 | 1-11 |
| 社交媒体 | 微信/朋友圈/微博/小红书 | 12-15 |
| 知识平台 | 知乎/知乎专栏/知乎盐选 | 16-20 |
| 视频平台 | B站/抖音/快手 | 21-23 |
| 资讯平台 | 网易号/百家号/搜狐号/大鱼号/企鹅号/一点号/今日头条/UC号 | 24-36 |

**原则**：所有发布均为"提示复制"，不自动发布到任何平台
```

## 修复优先级

**高优先级**：方案 A（精简版）
- 立即减少冗余，提升技能可读性
- 保持原有语义，不影响功能

**中优先级**：方案 B（结构化版）
- 需要更多重构工作
- 建议在后续会话中逐步实施
